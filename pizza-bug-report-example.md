При регистрации багов важно указывать следующую информацию:
# ID 123
# Не работает кнопка отмены заказа
## Описание
**Предусловия**: создан хотя бы один заказ

Тестовые данные - на каких тестовых данных нужно воспроизводить баг
Шаги воспроизведения - какие действия приводят к возникновению бага
Ожидаемый результат - какое поведение программы требуется, согласно требованиям
Наблюдаемый результат - (фактический результат) какое поведение программы зафиксировано во время тестирования
Найден в версиях - на какой версии (или версиях) системы обнаружен баг
Критичность (Severity, Серьёзность, Важность) - насколько большой ущерб наносит баг системе
Приоритет (Priority) - в каком порядке баги надо брать в работу
